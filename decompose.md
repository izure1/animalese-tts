한국어(한글)는 일본어나 영어와 달리 **초성, 중성, 종성이 결합된 '음절 문자'**이기 때문에, 동물의 숲 스타일의 TTS를 구현할 때 조금 더 복잡한 **분해(Decomposition)** 과정이 필요합니다.

개발자 관점에서 한글 입력을 발음으로 변환하는 단계를 분석해 보겠습니다.

---

## 1. 한글 처리의 핵심: 유니코드 분해

한글 한 글자는 하나의 유니코드 번호를 갖지만, 이를 발음하기 위해서는 초성(19개), 중성(21개), 종성(28개)으로 분해해야 합니다.

### 유니코드 계산 공식
한글 음절의 인덱스는 다음과 같은 수식으로 계산됩니다.
$$Index = ((초성 \times 21) + 중성) \times 28 + 종성$$
이를 역산하면 각 자음과 모음의 인덱스를 추출할 수 있으며, 이 인덱스를 미리 준비된 사운드 샘플(음소)에 매핑합니다.

---

## 2. 발음 합성 프로세스

### 1단계: G2P (Grapheme-to-Phoneme) 변환
단순히 글자대로 읽으면 어색합니다. 예를 들어 '국물'은 [궁물]로, '신라'는 [실라]로 발음되어야 자연스럽습니다.
* **적용:** 텍스트를 오디오로 변환하기 전, 한국어 음운 규칙(비음화, 유음화 등)을 적용하여 실제 발음 열로 변환합니다.

### 2단계: 음소 매핑 (Phoneme Mapping)
추출된 초/중/종성을 사운드 라이브러리와 연결합니다.
* **초성(자음):** 타격음처럼 짧게 재생됩니다.
* **중성(모음):** 목소리의 핵심 톤을 결정하며, 음절의 길이를 조절하는 구간입니다.
* **종성(받침):** 음절을 닫아주는 소리입니다. 받침이 없는 경우 '공백' 샘플이 들어갑니다.

### 3단계: 오디오 스트레칭 및 피치 시프팅
동물의 숲 특유의 느낌을 위해 **속도는 2~3배**, **피치는 1옥타브 이상** 높입니다. 이때 중요한 것은 **'포먼트 유지'**입니다. 단순히 피치만 올리면 금속음이 들릴 수 있으므로, 사람의 목소리 특징을 유지하며 주파수 영역을 이동시킵니다.

## 3. TypeScript 구현 예시 (음절 분해 로직)

한글 유니코드를 분석하여 각 음소의 샘플을 찾는 로직을 타입스크립트로 구성하면 다음과 같습니다.

```typescript
const CHOSEONG = ['ㄱ', 'ㄲ', 'ㄴ', 'ㄷ', 'ㄸ', 'ㄹ', 'ㅁ', 'ㅂ', 'ㅃ', 'ㅅ', 'ㅆ', 'ㅇ', 'ㅈ', 'ㅉ', 'ㅊ', 'ㅋ', 'ㅌ', 'ㅍ', 'ㅎ']
const JUNGSEONG = ['ㅏ', 'ㅐ', 'ㅑ', 'ㅒ', 'ㅓ', 'ㅔ', 'ㅕ', 'ㅖ', 'ㅗ', 'ㅘ', 'ㅙ', 'ㅚ', 'ㅛ', 'ㅜ', 'ㅝ', 'ㅞ', 'ㅟ', 'ㅠ', 'ㅡ', 'ㅢ', 'ㅣ']
const JONGSEONG = ['', 'ㄱ', 'ㄲ', 'ㄳ', 'ㄴ', 'ㄵ', 'ㄶ', 'ㄷ', 'ㄹ', 'ㄺ', 'ㄻ', 'ㄼ', 'ㄽ', 'ㄾ', 'ㄿ', 'ㅀ', 'ㅁ', 'ㅂ', 'ㅄ', 'ㅅ', 'ㅆ', 'ㅇ', 'ㅈ', 'ㅊ', 'ㅋ', 'ㅌ', 'ㅍ', 'ㅎ']

interface Phonemes {
  cho: string
  jung: string
  jong: string
}

function decomposeHangul(char: string): Phonemes | null {
  const code = char.charCodeAt(0) - 0xAC00
  if (code < 0 || code > 11171) return null

  const jongIndex = code % 28
  const jungIndex = ((code - jongIndex) / 28) % 21
  const choIndex = Math.floor((code - jongIndex) / 28 / 21)

  return {
    cho: CHOSEONG[choIndex],
    jung: JUNGSEONG[jungIndex],
    jong: JONGSEONG[jongIndex]
  }
}

// 사용 예시
const result = decomposeHangul('안')
console.log(result) // { cho: 'ㅇ', jung: 'ㅏ', jong: 'ㄴ' }
```

## 4. 비판적 검토 및 한계

**비판:** "종성(받침)을 모두 발음하면 속도가 너무 느려지거나 발음이 뭉개지지 않을까?"

**재검토 결과:**
실제 애니멀리즈 시스템에서는 종성을 생략하거나 아주 미세한 소리로 처리하는 경우가 많습니다. 한국어의 경우 **[초성 + 중성]** 위주로 소리를 내고, 종성은 뒤에 오는 초성과의 연결(연음 현상)을 위해 아주 짧게 처리하는 것이 '동물의 숲'스러운 옹알거림을 구현하는 팁입니다.

또한, **'ㅇ(이응)'** 초성의 경우 소리가 없으므로 모음 샘플만 바로 재생하여 속도감을 높이는 최적화가 들어갑니다.
